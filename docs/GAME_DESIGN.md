# GAME_DESIGN.md — ゲームデザイン仕様書

本ドキュメントがゲーム仕様の唯一の正（Single Source of Truth）である。
実装と食い違いが出た場合は本ドキュメントを優先し、変更する場合は本ドキュメントを先に更新する。

（仮題）: 『名もなき冒険者の生涯』 ※タイトルは後で決定。コード上は `roguelife` と呼ぶ。

---

## 1. コンセプト

- ジャンル: テキストベース・ローグライクRPG（横持ちスマホ、タップのみで遊べる）
- 一人の冒険者の「人生」を最初から最後まで体験する。死は永続。
- ダークで理不尽な世界。大半の選択肢に死がつきまとう。
- ただし死は無駄ではない。死ぬと「魂」が残り、来世（次のキャラ）を強くする。
- 世界観の語彙は一般的な異世界ファンタジー（ギルド、冒険者、ダンジョン、スライム等）。
  トーンはダークで乾いている。

### プレイの基本形

画面には常に**最大8つの選択肢**が提示される。プレイヤーはそこから1つをタップする。
それだけ。移動もフリック操作もない。すべての意思決定が8択（以下）に収まる。

### ゲームの構造（二層）

1. **人生レイヤー（1プレイ）**: キャラ作成 → 町で活動 → ダンジョンに挑む → 死 or 目的達成
2. **メタレイヤー（周回）**: 死亡・引退時に「魂」を精算 → 職業解放・来世への恩恵 → 新しい人生へ

## 2. キャラクター作成

1. **ステータスのダイスロール**: HP・力・素早さ・魔力・運 の初期値をダイスで決定する。
   振り直しは**最大3回まで**（4回目以降は不可。回数はUI上に残り回数として表示する）。
2. **年齢**: 全キャラクター共通で **18歳固定** からスタートする（ランダム決定は行わない）。
3. **職業の選択**: 所持している「魂」の数に応じて職業をアンロック済みなら選べる。
   一度アンロックした職業は永続的に選択可能。

### 職業（全5種）

| 職業 | 解放コスト(魂) | 特徴 |
|---|---|---|
| 無職 | 0 | 初期職。すべてが平凡。スキルなし |
| 剣士 | 15 | HP・力が高い。安定した近接戦闘 |
| 盗賊 | 30 | 素早さが高い。スキル「先制攻撃」「逃走」。盗みの成功率が上がる |
| 魔法使い | 60 | 魔力が高い。スキル「魔法攻撃」（敵の防御を無視） |
| 聖騎士 | 120 | 全体的に高水準。スキル「自己回復」。最上位職 |

※数値は初期案。`balance.ts` で調整可能にする。

## 3. 年齢と時間（コア経済）

本作の時間軸は「日」ではなく **年齢そのもの** を基本単位とする。
プレイヤーが選択肢（ボタン）を1つ選ぶたびに、その行動に応じた分だけ年齢が進む。
つまり**年齢は寿命の目盛りであると同時に、あらゆる行動の対価として支払われる
コスト資源**である。

### 3.1 基本原則

- キャラクターは **18歳** からスタートする（ダイスによるランダム決定は行わない）。
- 行動（ボタン選択）を1回実行するたびに、その行動固有の「年齢コスト」が加算される。
  年齢コストは行動ごとに異なり、小数（例: 0.3歳）も許容する。
- **リターンとコストは基本的に比例する**: 得られる金・ステータス上昇・アイテム価値が
  大きい行動ほど、年齢コストも大きく設計する。
  「効率よく歳を取らずに大きく稼ぐ」ことは基本的にできない。
- **例外: 突然死を伴うリスク行動は、年齢コストを意図的に低く設定する。**
  ロシアンルーレット、闘技場、危険な裏稼業などが該当する。これらは
  「死という別のコストを既に背負っている」ため、時間コストでは軽くする。
  つまりプレイヤーは常に「安全だが歳を取る道」と「速いが死にうる道」を
  天秤にかける構造になる。
- 加齢による衰え・老衰死のルール（セクション3.5）はそのまま維持し、
  上記の年齢コスト消費に対して適用される。

### 3.2 行動カテゴリ別の年齢コスト方針

| カテゴリ | 年齢コストの傾向 | 死亡リスク | 例 |
|---|---|---|---|
| 安全な労働・雑務 | 中〜高（リターンに比例して逓増） | なし | 労働（1〜10年） |
| 通常の稼ぎ・盗み | 中 | 低〜中（発覚時） | 盗み、行商、クエスト |
| 裏稼業・危険な仕事 | 低〜中 | 中 | 裏路地の仕事、脱獄 |
| **即死リスクのある賭け事・見世物** | **低（意図的に安い）** | **高（即死あり）** | ロシアンルーレット、闘技場 |
| ダンジョン探索・戦闘 | 行動ごとに小さく加算 | 中〜高 | 1地点の探索、1戦闘 |
| ダンジョン野営（眠る） | 中（従来の「1日」相当） | なし | 野営して深度+1 |
| 会話・情報収集など無リスク行動 | 極小 or ゼロ | なし | 噂を聞く、ステータス確認 |

- 上記は方針であり、具体的な年齢コスト値・リターン値は `balance.ts` に集約し、
  実装時に調整可能とする。
- 「安全なほど年齢コストが高く、危険なほど年齢コストが低い」という**逆相関**が
  本作の中心的な緊張構造である。この原則を崩す新規行動を追加する場合は、
  必ず本セクションの表に立ち返って整合性を確認すること。

### 3.3 労働（期間労働）との統合

セクション4の「労働」は、この年齢コスト経済の一形態として扱う。
労働は他の行動と異なり、**1回の選択で1〜10年分の年齢コストをまとめて消費する**
特殊な行動である（詳細はセクション4を参照）。年数が長いほど総報酬・年あたり効率が
逓増する設計は、上記3.1の「リターンとコストは比例する」の具体例として位置づけられる。

### 3.4 ダンジョンとの統合

従来「1日単位」としていたダンジョンの野営（眠る→深度+1）は、
年齢コストの一種として再定義する。野営1回あたりの年齢コストは、
町の「安全な労働」に近い中程度の値を初期案とする。ダンジョン内の探索・戦闘の
1アクションごとにも小さな年齢コストを加算し、「潜り続けるほど歳を取る」
プレッシャーを持たせる。

### 3.5 加齢による衰えと寿命（従来ルールを維持）

- **加齢による衰え**: 一定年齢（例: 45歳）を超えると、整数年齢が上がる（誕生日を越える）
  たびにステータスが減少していく。
- 寿命: 年齢が上限（例: 60歳＋ランダム）に達すると老衰死（＝引退扱いで魂精算）。
- この時間圧力により「悠長にレベル上げばかりはできない」構造を作る。

## 4. 町（ゲーム開始時の拠点）

プレイヤーは町で行動を開始する。町の行動もすべて8択以下の選択肢で表す。
町での主要な行き先（イベント発生地点）:

| 行き先 | できること |
|---|---|
| 冒険者ギルド | クエスト受注（報酬つき討伐・採取）、ランク確認 ※ギルドランクは将来のゴール用 |
| 酒場 | 情報収集（噂＝攻略ヒント）、ギャンブル（カード、ロシアンルーレット等） |
| 武器屋・防具屋 | 購入 / **盗む**（成功で入手、失敗で戦闘 or 牢屋） |
| 道具屋 | ポーション、薬草、帰還の巻物などの購入 / 盗み |
| 裏路地 | 裏稼業の斡旋（運び屋、脅し、暗殺の手伝い等。高報酬・高リスク。失敗で牢屋 or 死） |
| 労働 | 安全に金を稼ぐ。1回の選択で1〜10年の年齢をまとめて消費（下記「労働（期間労働）」参照） |
| 教会 | 回復（有料）、呪いの解除 |
| ダンジョン入口 | ダンジョンへ入る（→ セクション5） |

- **牢屋**: 盗み・裏稼業の失敗で投獄。年齢を消費して出所するか、脱獄を試みる
  （脱獄失敗は死につながる）。
- 金（ゴールド）は死んだら失われる。魂だけがメタ資産。

### 労働（期間労働）

- 労働は1回の選択で **1〜10年** の契約期間を選び、その年数分の年齢コストをまとめて
  消費して報酬を一括で受け取る。
- 年数が長いほど**総報酬も年あたり効率も逓増**する（セクション3.1「リターンとコストは
  比例する」の具体例）。
- 死亡リスクはない。安全な代わりに、人生で最も年齢を重く支払う稼ぎ方である。

## 5. ダンジョン（コアコンテンツ、最初に実装する）

『異世界の闇の中で』方式を採用する。

### 基本ループ

1. ダンジョンに入ると、**8つの「未探索地点」**が提示される。
2. 1つを選ぶと、その先の内容が判明する（敵、宝箱、罠、泉、行商人、野営地 など）。
3. 目標は各フロアで**「野営地」（安全地帯）を見つけて眠る**こと。
4. 眠ると年齢コストを消費し（セクション3.4）、**ダンジョンが1段階深くなる**
   （＝次の「フロア」へ）。
   深くなるほど敵は強く、宝はより良くなる。
5. フロア構成は毎回自動生成（乱数）。

### 深度スケーリング

- 深度 d に応じて敵のステータス・出現テーブル・ドロップ品質が上昇する。
- 一定深度ごとに**中ボス**が出現し、その先に**最深部のボス**（v1の最終目的）を置く。
- v1 のボスは深度10〜15程度に置く（バランス調整で決定）。

### 帰還は困難（重要デザイン）

一度ダンジョンに入ったら、町に戻るのには労力がかかる。

- 帰還手段は次のいずれか:
  1. **歩いて戻る**: 現在の深度ぶんのフロアを、1フロアごとに年齢コストを消費して遡る。
     遭遇は発生し続ける（敵の強さはそのフロア相当）。
  2. **帰還の巻物**: 高価な消費アイテム。即座に町へ戻れる。町の道具屋・ダンジョン内の
     行商人で稀に入手可能。
- つまり「もう1日だけ深く潜るか、ここで引き返すか」が常に緊張を生む。

### ダンジョン内の遭遇（初期実装セット）

- 敵との戦闘（ターン制コマンドバトル: 攻撃 / スキル / アイテム / 逃走）
- 宝箱（罠つきのことがある。解除失敗でダメージ〜即死）
- 泉（回復 or 毒。見た目では区別できないが、酒場の噂でヒントが得られる）
- ゴミの山（漁るとアイテム or 何か良くないこと）
- 行商人（購入 / 盗む）
- 罠（気づけるかは素早さ・運に依存）
- 野営地（眠って次のフロアへ / その場で休息して回復のみ、の選択）
- 稀にレアモンスター（撃破で魂ボーナス）

## 6. 戦闘

- **ターン制コマンドバトル**。コマンドは最大4つ: 攻撃 / スキル / アイテム / 逃走。
- 行動順は素早さで決まる。盗賊の「先制攻撃」は必ず先手。
- ダメージ計算は単純に: `damage = max(1, attack - defense) * variance`。
  魔法攻撃は defense を無視する。
- 逃走は素早さと運で成功率が決まる。失敗するとそのターンは無防備。
- HPが0になったら**死**。復活はない。

## 7. 死と魂（メタ進行の核）

### 死亡・引退時の魂精算

- 冒険者の人生が終わる（死亡 / 老衰 / 自主引退）と、その生涯の実績に応じて**魂**を獲得する。
- **重要ルール: 獲得できる魂には、その人生の「目的達成度」による上限がある。**

| 達成度ティア | 条件（例） | 魂の獲得上限 |
|---|---|---|
| 0: 野垂れ死に | 特に何も成し遂げていない | 5 |
| 1: 駆け出し | ダンジョン深度3到達 or 所持金500G到達 | 15 |
| 2: 一人前 | 深度6到達 or 中ボス撃破 or 2000G到達 | 40 |
| 3: 熟練 | 深度10到達 or 中ボス2体撃破 | 80 |
| 4: 英雄 | 最深部のボス撃破（v1のクリア） | 上限なし |

- ティア内での実際の獲得量は、生きた年数・撃破数・所持金などから計算し、上限でキャップする。
- 自主引退は「教会」または「ギルド」から行える（獲得魂に若干のボーナス）。

### 来世への恩恵（レガシー）

特定の目的を達成すると、以後のすべてのキャラが恩恵を受ける（永続アンロック）:

- 例: 中ボス初撃破 → 来世から初期装備に「鉄の剣」が加わる
- 例: 累計10回ダンジョン深度5到達 → 道具屋の価格が10%引き
- 例: ボス撃破 → 新しい難易度 or 新シナリオの解放（将来フェーズ）

レガシーはデータ駆動で追加できる設計にする（`src/data/legacies.ts`）。

## 8. 最終ゴール（マルチエンディング構想）

- **v1（最初のリリース目標）: ダンジョン最深部のボス討伐** のみ実装。
- 将来フェーズで追加するゴール（設計だけ先に考慮しておく）:
  - 商人ルート: 一定の資産を築き、町の有力者になる
  - 裏稼業ルート: 裏社会で成り上がる
  - ギルドルート: ギルドランク最上位に到達する
- ゴール達成でその人生は「大団円」として終わり、大量の魂とレガシーを得る。
- 実装上は「ゴール」をデータとして定義し（達成条件・報酬・エンディングテキスト）、
  追加がデータ追記で済むようにする。

## 9. UI 方針

- **横持ち（ランドスケープ）固定**。黒背景＋白文字基調のロートーン。
- メイン画面は**横に3分割**する:
  - **左右**: 選択肢ボタンの枠を縦に4つずつ配置（計8スロット）。場面ごとに最大8つの
    選択肢を割り当てる。その場面で使わないスロットは**空白のまま表示**する
    （枠は残し、レイアウトは常に固定）。
  - ボタンは縦に小さめ（高さ固定）にし、**カラム上部は空けておく**。
  - **中央**: 上部に場面を描写するイラスト、その下にテキストログ。
    比率は**イラスト7 : テキスト3**。
- **右上隅にシステムボタン**を置く: ステータス詳細 / 中断 / 設定。
- ステータス（HP/金/年齢/深度）は中央カラム最上部に細い帯で常時表示する。
- テキストは1画面に収まる分量で区切り、タップで送る。
- 画像はGemini生成のものを差し込む（未支給時はプレースホルダー）。

## 10. セーブ

- 人生の途中経過は自動セーブ（アプリを閉じても再開できる）。
- **セーブ&ロードによる死の回避（リセマラ）は不可**にする: 行動決定時に即時セーブする。
- メタ進行（魂、アンロック、レガシー、統計）は別領域に保存。

- 買い切り ¥120 で広告除去
