# ROADMAP.md — 開発フェーズ計画

このフェーズ順に実装する。各フェーズの「完了条件」をすべて満たしてから次へ進むこと。
フェーズ外の機能を先取りしないこと。

---

## フェーズ0: プロジェクト土台

やること:
- Expo (TypeScript) プロジェクトの新規作成
- ディレクトリ構成の作成: `src/core/` `src/ui/` `src/data/` `src/data/texts/`
- ESLint / Prettier / テストランナーの設定
- シード可能な乱数モジュール（`src/core/rng.ts`）
- ゲーム状態の型定義の骨格（`src/core/types.ts`）
- 黒背景＋白文字の基本テーマと「テキストログ＋選択肢ボタン」の共通UIコンポーネント

完了条件:
- スマホ（Expo Go）で起動し、ダミーの選択肢を押すとログが増える画面が動く
- `npm test` が通る（rngのテスト最低1本）
- ユーザーがスマホで動作確認する手順書を提示した

## フェーズ1: キャラ作成と町の基本ループ

やること:
- キャラ作成フロー: ステータスのダイスロール（振り直しは最大3回）→ 職業選択
  （この時点では「無職」のみ。年齢は18歳固定）
- 町のハブ画面: 8択で行き先を選ぶ
- 最小の町コンテンツ: 労働（期間1〜10年、金と年齢コスト）、道具屋（購入のみ）、
  酒場（噂のダミー）
- 年齢コストによる時間進行、加齢による衰え、老衰死（GAME_DESIGN.md セクション3）
- 自動セーブ（人生の途中経過の保存と再開）

完了条件:
- キャラを作り、労働で金を稼ぎ、年齢を消費して老衰死するまでを通しでプレイできる
- 年齢減衰・年齢進行のユニットテストが通る

## フェーズ2: ダンジョンと戦闘（コアコンテンツ）

やること:
- ダンジョン入口（町の行き先に追加）
- 分岐マップ方式のフロア自動生成（内部ノードグラフ・非表示、進行は最大3択）
- 遭遇の初期セット: 敵 / 宝箱（罠つき含む）/ 泉 / 何もない空間 / 野営地
- ターン制戦闘（攻撃 / アイテム / 逃走。スキルは職業実装後）
- 野営地で眠る → 深度+1 → 敵・宝のスケーリング。常設ボタン（アイテム・引き返す）
- 帰還システム: 歩いて戻る（フロア遡行）/ 帰還の巻物
- 死亡処理（人生の終了画面。魂精算はまだ仮でよい）

完了条件:
- 町→ダンジョン→数フロア潜る→帰還 or 死亡、の一連が通しで遊べる
- 戦闘計算・深度スケーリングのユニットテストが通る
- ここで一度ユーザーにプレイしてもらい、手応えのフィードバックを得る

## フェーズ3: 魂とメタ進行

やること:
- 死亡・引退時の魂精算（達成度ティアによる上限、GAME_DESIGN.md セクション7）
- 魂の永続保存、周回フロー（死 → 精算画面 → 新しいキャラ作成）
- 職業5種の実装（魂によるアンロック、職業ごとのステ補正とスキル）
- スキルを戦闘コマンドに追加（先制攻撃、逃走強化、魔法攻撃、自己回復）
- レガシー（来世への恩恵）の仕組みとデータ定義（最初は2〜3個でよい）
- 自主引退（教会/ギルドから）

完了条件:
- 死んで魂を得て、職業を解放し、強くなって再挑戦する周回ループが成立している
- 魂精算（上限キャップ含む）のユニットテストが通る

## フェーズ4: 町コンテンツとリスク行動の拡充

やること:
- 盗み（武器屋・防具屋・道具屋・行商人）: 成功/戦闘/牢屋の分岐
- 牢屋と脱獄
- 裏路地の裏稼業（2〜3種類）
- 酒場のギャンブル（カード等1種 + ロシアンルーレット）
- 酒場の噂システム（泉の見分け方などの実用ヒントをランダム提供）
- 冒険者ギルド: 簡単なクエスト受注（討伐・採取）と報酬
- ダンジョン遭遇の追加: ゴミの山、行商人、罠、レアモンスター

完了条件:
- 「労働 / 盗み / 裏稼業 / ギャンブル / クエスト」の5系統で金が稼げ、それぞれに固有の
  リスク（牢屋・死）がある
- イベントがマスターデータ追記で増やせることを、実際に1件追加して示す
